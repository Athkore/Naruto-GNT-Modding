Instructions in this opcode group seem to be related to branching instructions.

Uses:

- Count Register (`ctr`) = reg_p[0x14]
- Comparison Register (`cr`) = reg_p[0x15]
  - Values are set to this in opcode group 01, 04, 05, 06, and 07.
- Stack Pointer (`sp`) = reg_p[0x17]
  - Used in opcode group 01 for return addresses of subroutine calls

## 013B - bdnz

Branch decrement not zero.

Branch if `ctr` is less than 0. Otherwise, decrement `ctr` and if `ctr` is then not 0, branch. This is used for looping. For example, load 100 in `ctr` and using this opcode you can branch backwards to loop 100 times.

- `param1`: The offset to branch to.

## 013C - bl

## 014E

- Parameters: 0 or 1 (Based on `SEQ_RegCMD1`)

## 014F - blrdnz

Branch and link return decrement not zero.

Branch and link return if `ctr` is less than 0. Otherwise, decrement `ctr` and if `ctr` is then not 0, branch and link return. This is used for looping. For example, load 100 in `ctr` and using this opcode you can return from the current subroutine 100 times; after 100 times you won't return and will then run anything after this opcode.

- Parameters: 0

## 0150

Some kind of switch-case opcode.

Call `SEQ_RegCMD1` and return `returnVal`. If `returnVal` is less than 0 OR `pc` is less than or equal to `returnVal` OR the value at `pc + 1` with offset `returnVal` is equal to zero: add 1 to `pc` and then add the value at the new `pc`. Otherwise, set `pc` to `seq_p->seq_p_sp[0x17]` + `returnVal`.

There is an array of seq offsets after this opcode.

- Parameters: Variable based on `SEQ_RegCMD1` and parameter
